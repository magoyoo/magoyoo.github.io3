Swift 고급 오류 처리 전략: 열거형과 커스텀 오류를 사용한 견고한 오류 처리 시스템 구축

Swift는 강력한 오류 처리 메커니즘을 제공합니다. 여기에 열거형과 커스텀 오류를 활용하면 더욱 견고하고 유지보수하기 쉬운 코드를 작성할 수 있습니다. 이 글에서는 Swift의 오류 처리 방법에 대해 깊이 있게 탐구하고, 열거형과 커스텀 오류를 사용하여 복잡한 시나리오에서도 문제가 발생하지 않도록 하는 방법을 설명합니다.

기본 오류 처리

Swift에는 내장된 오류 처리 모델이 있습니다. 가장 기본적인 방법은 throw, try, catch 키워드를 사용하는 것입니다.

swift
enum FileError: Error {
    case fileNotFound
    case unreadable
    case encodingFailed
}

func readFile(fileName: String) throws -> String {
    // 파일을 읽는 로직
}

func example() {
    do {
        let content = try readFile(fileName: "example.txt")
        print(content)
    } catch FileError.fileNotFound {
        print("파일을 찾을 수 없습니다.")
    } catch {
        print("알 수 없는 오류 발생.")
    }
}

위 코드는 FileError 열거형을 사용하여 파일 처리 중 발생할 수 있는 다양한 오류를 정의합니다. readFile 함수에서는 발생 가능한 오류를 throws 키워드로 표시하며, 이를 호출하는 example 함수에서는 do-catch 블록을 사용하여 오류를 처리합니다.

이 기본적인 구조를 사용하면 대부분의 일반적인 오류 처리는 가능하지만, 더 복잡한 사례를 위해서는 추가적인 방법이 필요합니다.

열거형을 활용한 고급 오류 처리

열거형을 활용하면 오류 상황을 매우 구체적이고 명확하게 정의할 수 있습니다. 이는 코드를 읽고 이해하는 데 중요한 도움을 줍니다.

기본 열거형 오류

일반적인 열거형을 사용한 오류 처리는 아래와 같습니다:

swift
enum NetworkError: Error {
    case badURL
    case requestFailed(description: String)
    case unknown
}

func fetchData(urlString: String) throws {
    guard let url = URL(string: urlString) else {
        throw NetworkError.badURL
    }

    // 네트워크 요청 로직, 여기서 requestFailed나 unknown 오류를 던질 수 있음
}

커스텀 오류 정의

열거형을 사용하여 커스텀 오류를 정의하면 오류의 유형을 더 세분화할 수 있습니다. 이에 따라 특정 상황에 맞는 오류 메시지를 제공할 수 있습니다.

swift
enum DatabaseError: Error {
    case connectionFailed(reason: String)
    case queryFailed(query: String, description: String)
    case unknown
}

func fetchUser(id: Int) throws -> User {
    guard id > 0 else {
        throw DatabaseError.queryFailed(query: "SELECT * FROM users WHERE id=-1", description: "ID는 0보다 커야 합니다.")
    }
    // 데이터베이스 로직, 여기서 connectionFailed나 unknown 오류를 던질 수 있음
}

커스텀 오류와 열거형을 통한 확장성

고급 오류 처리를 위해 열거형과 커스텀 오류를 함께 사용하면 코드를 확장하기 쉽습니다. 예를 들어, 네트워크 라이브러리를 작성하는 경우 다양한 오류 상황을 미리 정의해두면 나중에 같은 라이브러리를 사용하는 다른 코드에서도 보다 쉽게 오류를 처리할 수 있습니다.

swift
enum AppError: Error {
    case networkError(NetworkError)
    case databaseError(DatabaseError)
    case generalError(description: String)
}

func performOperation() {
    do {
        let result = try someRiskyOperation()
        // 결과 처리
    } catch let error as NetworkError {
        throw AppError.networkError(error)
    } catch let error as DatabaseError {
        throw AppError.databaseError(error)
    } catch {
        throw AppError.generalError(description: error.localizedDescription)
    }
}

위 코드에서, performOperation 함수는 네트워크와 데이터베이스 관련 오류를 각각 NetworkError와 DatabaseError로 캡슐화하여 보다 구체적인 오류 정보를 호출자에게 전달합니다. 이렇게 하면 각 오류 상황에 대한 구체적인 처리가 가능합니다.

결과 타입(Result Type)을 활용한 오류 처리

최근 Swift에서는 결과 타입(Result Type)도 많이 사용됩니다. 이는 함수가 성공했는지, 실패했는지를 명확하게 표시할 수 있는 방법입니다.

swift
enum LoginError: Error {
    case invalidCredentials
    case userNotFound
    case serverError(description: String)
}

func login(username: String, password: String) -> Result<String, LoginError> {
    // 로그인 로직
    // 성공 시: .success("토큰")
    // 실패 시: .failure(.invalidCredentials) 또는 다른 오류 유형
}

let result = login(username: "user", password: "password")

switch result {
case .success(let token):
    print("로그인 성공, 토큰: \(token)")
case .failure(let error):
    switch error {
    case .invalidCredentials:
        print("유효하지 않은 자격 증명")
    case .userNotFound:
        print("사용자를 찾을 수 없습니다.")
    case .serverError(let description):
        print("서버 오류: \(description)")
    }
}

결과 타입을 사용하면 함수의 반환값을 통해 오류를 검사할 수 있으므로 코드의 가독성이 높아집니다.
